## 1. Введение и мотивация (20 минут)

### История и рождение Linux (7 минут)
- **Ключевые персоны:** Линус Торвальдс, 1991 год, студент из Финляндии
- **Мотивация:** "Делаю бесплатную ОС для 386(486) AT клонов"
- **Эволюция:** От Unix → Minix → Linux
- **Философия Open Source:** код открыт для всех, каждый может улучшить

### "Вы любите бесплатное ПО?" - Преимущества открытого кода (8 минут)
- **Бесплатность:** Не нужно пиратить, все легально и бесплатно
- **Безопасность:** Тысячи глаз видят все баги (Linus's Law)
- **Кастомизация:** Можно изменить что угодно под свои нужды
- **Обучение:** Можно изучить, как все работает "под капотом"
- **Что значит "открытое ПО":** исходный код vs скомпилированная программа

### Текущее положение на рынке ОС (5 минут)
**Windows проблемы:**
- Легаси: интерфейсы 2007 года под новыми плашками
- Bloatware: куча предустановленного мусора
- Телеметрия и AI-функции сомнительной пользы
- Но: можно почистить за час-два, WSL для разработки

**Linux преимущества:**
- Производительность: Steam Deck на SteamOS (Linux)
- Удобство разработки: пример установки Python библиотек
- Стабильность серверов: 90%+ интернета работает на Linux

## 2. Знакомство с терминалом (25 минут)

### Что такое терминал и зачем он нужен (5 минут)
- **Определение:** прямой интерфейс с операционной системой
- **Преимущества:** скорость, автоматизация, точность
- **Мифы:** "это только для хакеров" vs реальность
- **Запуск:** `Ctrl+Alt+T`, внешний вид: prompt, курсор

### Анатомия команды (10 минут)
```bash
команда [флаги] [аргументы]
ls -la /home/user
│   │   │
│   │   └── аргумент (что обработать)
│   └── флаг (как обработать)  
└── команда (что делать)
```

**Ключевые концепции:**
- Команды = программы в системе
- Флаги изменяют поведение (`-l`, `-a`, `--help`)
- Автодополнение `Tab`, история `↑↓`
- `man команда` - встроенная справка

### Базовые команды навигации (10 минут)
```bash
pwd          # Print Working Directory - где я?
ls           # List - что здесь?
ls -l        # подробная информация (права, размер, дата)
ls -a        # показать скрытые файлы (начинающиеся с .)
ls -la       # комбинация флагов
cd путь      # Change Directory - перейти
cd ..        # на уровень вверх  
cd ~         # домашняя папка
cd /         # корень системы
```

**Практика:** Исследование файловой системы, понимание иерархии

## 3. Работа с файлами и папками (20 минут)

### Создание и управление (10 минут)
```bash
mkdir название                    # создать папку
mkdir -p путь/до/вложенных/папок # создать структуру
touch файл.txt                   # создать пустой файл
cp источник назначение           # копировать
cp -r папка/ новое_место/        # копировать папку рекурсивно
mv старое_имя новое_имя          # переименовать/переместить
rm файл                          # удалить файл
rm -r папка/                     # удалить папку с содержимым
rm -rf папка/                    # принудительное удаление
```

### Чтение и редактирование файлов (10 минут)
```bash
cat файл.txt              # вывести содержимое целиком
less файл.txt             # постраничный просмотр (q для выхода)
head -n 5 файл.txt        # первые 5 строк
tail -n 3 файл.txt        # последние 3 строки
echo "текст" > файл.txt   # записать в файл (перезапись)
echo "текст" >> файл.txt  # дописать в файл
nano файл.txt             # простой текстовый редактор
```

**Практическое задание:** Создание структуры папок "Школа" с предметами и файлами

## 4. Продвинутые возможности (15 минут)

### Поиск и фильтрация (7 минут)
```bash
find . -name "*.txt"              # найти все .txt файлы
find /home -type d -name "Download*" # найти папки начинающиеся с Download
grep "слово" файл.txt             # найти строки со словом
grep -r "слово" папка/            # рекурсивный поиск в папке
which команда                     # где находится программа
locate файл                       # быстрый поиск по базе данных
```

### Перенаправление ввода-вывода (8 минут)
```bash
команда > файл                    # перенаправить вывод в файл
команда >> файл                   # дописать вывод в файл
команда < файл                    # взять ввод из файла
команда1 | команда2               # pipe - вывод первой во вторую
ls -la | grep "txt"               # найти только .txt файлы в списке
cat файл.txt | wc -l              # посчитать строки в файле
```

**Полезные комбинации:**
- `history | grep "mkdir"` - найти все использования mkdir
- `ps aux | grep firefox` - найти процессы Firefox
